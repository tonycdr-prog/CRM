This is a classic “scroll is trapped” issue: the Forms page (or a parent layout) is likely using `overflow-hidden` or a fixed-height container without `overflow-y-auto`, so when the template renders, the page can’t scroll.

Fix it in the MOST reliable way: make the Forms page’s main content area explicitly scrollable.

✅ Change: client/src/pages/field-job-forms.tsx

1) Wrap the whole page content in a fixed viewport-height container with a scrollable inner area.
2) Ensure the scrollable div has `min-h-0` (important in flex layouts) and `overflow-y-auto`.
3) Add some bottom padding so the last fields aren’t hidden behind any fixed UI.
